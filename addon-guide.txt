Java & Bedrock Import Guide: How to Install Minecraft Add‑ons, Mods, and Maps

## 1. Introduction
Minecraft has two main editions: Bedrock (mobile, Windows 10/11, console) and Java (Windows, macOS, Linux). The way add‑ons, mods, and custom content are installed is completely different between them. This guide covers both, step by step.

Bedrock uses official auto‑import file formats (.mcpack, .mcaddon, .mcworld, .mcskin). Simply opening these files imports them into the game. Java requires manual file placement in specific folders inside the .minecraft directory and often depends on mod loaders like Forge or Fabric.

Always check which edition your content was made for — using the wrong edition’s file will result in it not working.

## 2. Bedrock Edition — Supported File Types
Bedrock add‑ons come in specific file extensions. These are:

- .mcpack — A single resource pack (textures, sounds, UI) OR a single behaviour pack (game mechanics, mobs, items). Tapping it imports it directly.
- .mcaddon — A bundle containing BOTH a resource pack and a behaviour pack together. When imported, two separate packs appear and both must be activated.
- .mcworld — A full world/map file. Importing creates a new world in your worlds list.
- .mcskin — A skin pack. Tapping opens the Dressing Room and adds the skin to your collection.

Important: If a downloaded file has a .zip extension instead of .mcpack/.mcaddon, simply rename the file extension (remove .zip and make it .mcpack or .mcaddon) before trying to open it.

## 3. Bedrock — Step‑by‑Step Installation

### Step 1: Verify File Types
Before doing anything, confirm the file extension is one of the supported types. Never open .exe, .apk, or other suspicious files.

### Step 2: Download the File
Download the add‑on from a trusted source. Recommended: MCPEDL, the MIA Discord server, CurseForge Bedrock section, or directly from creators. The file usually goes to your device’s Downloads folder.

### Step 3: Import the File
Open your device’s file manager (ZArchiver on Android is highly recommended). Navigate to the Downloads folder. Tap the .mcpack or .mcaddon file. If prompted, choose “Minecraft” to open it. If nothing happens, use “Open with” and select Minecraft.

If the file is .zip, rename it first: remove “.zip” and ensure the extension is .mcpack or .mcaddon.

Minecraft will launch and display:
“Import Started…” → then shortly after “Successfully Imported”.

### Step 4: Activate the Content
After importing, the pack still needs to be activated in your world:
- Create or edit a world.
- Scroll down to “Behaviour Packs” or “Resource Packs”.
- Tap the imported pack to move it to the “Active” column.
- For .mcaddon files, activate BOTH the behaviour pack AND the resource pack separately.
- Some add‑ons require Experimental Gameplay. Go to World Settings → Game → Experiments and enable the required toggles.

### Step 5: Worlds and Skins
- .mcworld files: After import, the world appears in your worlds list — just tap to play.
- .mcskin files: After import, go to Dressing Room → Owned Skins to apply the skin.

## 4. Bedrock Troubleshooting
- Import failed: Try “Open with” → Minecraft. Rename file extension to .mcpack/.mcaddon if it ended up as .zip. On Android, ensure Minecraft has storage permission. Restart device.
- Pack not showing in world: Activate both resource and behaviour packs for .mcaddon. Enable Experimental Gameplay. Check Minecraft version compatibility.
- File won't open: Use ZArchiver or a dedicated file manager. If the file is .zip, rename it as described.

## 5. Java Edition — Supported File Types
Java Edition content comes in these formats:
- .zip — Resource pack OR datapack. Do NOT extract.
- .jar — Mod file. Requires a mod loader (Forge, Fabric, NeoForge, Quilt).
- World folder — Extracted folder containing level data, placed in saves.

Java does not auto‑import; you must manually place files in the correct .minecraft subfolders.

## 6. Java — Resource Packs
Resource packs change textures, sounds, and UI. Installation:
1. Download the .zip file.
2. Open Minecraft → Options → Resource Packs → “Open Pack Folder”.
3. Drag the .zip into the folder that opens.
4. In the game, move the pack to the “Active” side.

Paths:
- Windows: %appdata%\.minecraft\resourcepacks
- macOS: ~/Library/Application Support/minecraft/resourcepacks
- Linux: ~/.minecraft/resourcepacks

Troubleshooting: Ensure the .zip is directly in the folder (not a subfolder). Check compatibility with your Minecraft version. Disable other packs to test.

## 7. Java — Mods (Forge / Fabric)
Mods require a mod loader. The most common are Forge and Fabric. You must install the loader that matches your Minecraft version first.

Installation steps:
1. Download and install the mod loader (Forge or Fabric) for your Minecraft version.
2. Locate the .minecraft folder:
   - Windows: Win+R → %appdata%\.minecraft
   - macOS: Cmd+Shift+G → ~/Library/Application Support/minecraft
   - Linux: ~/.minecraft
3. Inside .minecraft, create a folder called “mods” (if it doesn't already exist).
4. Place the downloaded .jar mod file into the mods folder.
5. Launch Minecraft using the Forge/Fabric profile.

Troubleshooting:
- Mod won't load: Check that the mod's version and loader match exactly.
- Crashes: Read the crash report; it often lists missing dependencies or incompatible mods.
- Dependencies: Many mods need other mods (e.g., Fabric API, Cloth Config). Always check the mod's page for required dependencies.

## 8. Java — Datapacks
Datapacks add custom recipes, advancements, functions, and more to specific worlds.
1. Download the datapack as .zip (do not extract).
2. Navigate to .minecraft/saves/[your world]/datapacks.
3. Place the .zip file there.
4. In game, run /reload or re‑enter the world.

## 9. Java — Worlds / Maps
Custom maps are distributed as folders.
1. Download the world (often .zip).
2. Extract the folder.
3. Move the extracted folder into .minecraft/saves.
4. The world appears in the singleplayer menu.

## 10. General Safety Tips
- Only download from trusted sources (MIA, MCPEDL, CurseForge, Modrinth, official websites).
- Avoid .exe, .apk, .msi files disguised as Minecraft content — these could be malware.
- Scan any suspicious file with antivirus before opening.
- For Java mods, always check compatibility (version, loader, dependencies) before installing.

## 11. Frequently Asked Questions
Q: Can I use Bedrock add‑ons on Java?
A: No. .mcpack and .mcaddon files only work on Bedrock Edition. Java uses .jar and .zip files.

Q: How do I know if a mod is for Fabric or Forge?
A: Check the mod's download page. It will specify which loader it requires. They are not interchangeable.

Q: What if my file downloaded as .zip but should be .mcpack?
A: Rename the extension from .zip to .mcpack and then open it with Minecraft.

Q: Do I need to extract .zip resource packs?
A: No. For Java resource packs, leave the .zip intact. Datapacks also stay as .zip.

Q: What is the mods folder?
A: Inside .minecraft, the mods folder is where you place .jar mod files. Create it if it doesn't exist.

Q: How do I get Forge or Fabric?
A: Download from the official sites: files.minecraftforge.net or fabricmc.net. Always match your Minecraft version.

## 12. Summary
- Bedrock: .mcpack / .mcaddon → tap to import → activate in world.
- Java: .jar → place in mods folder; .zip → place in resourcepacks or datapacks folder; world folder → place in saves.
- Always check edition compatibility, version, and loader requirements.
- Use MIA guides for detailed help on specific platforms.
